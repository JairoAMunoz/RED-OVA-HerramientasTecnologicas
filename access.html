Access

Aplicación de Microsoft Access

Uso de Access para la administración de los datos

Introducción

Hoy en día, la aplicabilidad de herramientas informáticas para el procesamiento de datos y la gestión de la información en diversos contextos, permite mayor efectividad en el tratamiento de la información. Este avance a nivel informático, ha permitido agilizar procesos que antes tomaban más tiempo y que ahora se simplifican con el uso efectivo de herramientas como Microsoft Access. Por esta razón, este avance exige a todo individuo el desarrollo de habilidades en el manejo de estas herramientas, para así estar en la capacidad de plantear procedimientos informáticos necesarios para la gestión de los datos y de la información de una entidad.

Actividad

Taller: Aprender a crear una base de datos sencilla en Access, incluyendo tablas, relaciones, formularios, consultas e informes.

Pasos de la actividad:
1. Crear la base de datos

  • Abrir Microsoft Access.
    Crear una nueva base de datos con el nombre Biblioteca.accdb.

2. Crear la tabla de Libros

Nombre de la tabla: Libros.
Campos:

ID_Libro (Clave primaria, Autonumérico).
Titulo (Texto corto).
Autor (Texto corto).
Año_Publicación (Número).
Categoría (Texto corto).
Disponible (Sí/No).

3. Crear la tabla de Estudiantes

Nombre de la tabla: Estudiantes.
Campos:

ID_Estudiante (Clave primaria, Autonumérico).
Nombre (Texto corto).
Apellido (Texto corto).
Curso (Texto corto).
Teléfono (Texto corto).

4. Crear la tabla de Préstamos

Nombre de la tabla: Prestamos.
Campos:

ID_Prestamo (Clave primaria, Autonumérico).
ID_Libro (Número).
ID_Estudiante (Número).
Fecha_Prestamo (Fecha/Hora).
Fecha_Devolución (Fecha/Hora).

5. Establecer relaciones

Libros.ID_Libro → Prestamos.ID_Libro.
Estudiantes.ID_Estudiante → Prestamos.ID_Estudiante.

6. Crear un formulario sencillo

7. Formulario para ingresar nuevos estudiantes.

Formulario para registrar nuevos libros.
Diseñar una consulta

8. Crear una consulta que muestre todos los libros prestados y el estudiante que los tiene.


9. Generar un informe

Crear un informe con la lista de préstamos activos, mostrando: título del libro, nombre del estudiante y fecha de devolución.

Criterios de Evaluación
Criterio Excelente (5 pts) Bueno (4 pts) Aceptable (3 pts) Deficiente (1-2 pts) Puntaje Máx.
Creación de la base de datos Base de datos creada correctamente con el nombre indicado. Base de datos creada con nombre diferente o detalles menores. Base de datos creada, pero con errores en configuración inicial. No se crea la base de datos correctamente. 5
Tablas Libros, Estudiantes y Préstamos Tablas creadas con todos los campos, tipos de datos y claves primarias correctamente. Tablas creadas con la mayoría de campos correctos, pequeños errores en tipos de datos. Tablas incompletas o con errores significativos en los campos. No se crean las tablas solicitadas o son incorrectas. 20
Relaciones entre tablas Relaciones creadas correctamente con integridad referencial. Relaciones creadas con pequeños errores. Relaciones incompletas o mal configuradas. No se crean relaciones. 15
Formularios Formularios de estudiantes y libros creados, funcionales y bien diseñados. Formularios creados, con diseño básico pero funcionales. Formularios incompletos o con errores de funcionamiento. No se crean formularios. 15
Consulta Consulta creada correctamente mostrando libros prestados y estudiantes. Consulta creada con pequeños errores en campos o criterios. Consulta incompleta o con resultados incorrectos. No se crea consulta. 15
Informe Informe de préstamos activos creado correctamente, con diseño claro y datos solicitados. Informe creado, con diseño básico pero información correcta. Informe incompleto o con errores en la presentación de datos. No se crea informe. 15
Presentación y organización Base de datos ordenada, con nombres adecuados en tablas, formularios, consultas e informes. Base de datos ordenada con algunos nombres genéricos. Base de datos poco organizada o con inconsistencias de nombres. Base de datos desordenada, difícil de entender. 15
